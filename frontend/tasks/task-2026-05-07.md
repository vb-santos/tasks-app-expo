# 📝 Desafio Prático: Gerenciamento Global de Estado com Zustand

Seja bem-vindo(a) a mais um desafio prático de React Native! O objetivo desta atividade é evoluir a arquitetura do aplicativo de tarefas, substituindo o gerenciamento de estado local e o repasse excessivo de propriedades (*prop drilling*) por uma solução robusta e moderna de estado global utilizando a biblioteca **Zustand**.

## 🎯 Tópicos Abordados

Neste exercício, você terá contato direto com os seguintes conceitos fundamentais:

- **Gerenciamento Global de Estado:** Criação de *stores*, *actions* (ações) e *selectors* (seletores).
- **Zustand:** Instalação e configuração de uma store simples e performática.
- **Refatoração de Arquitetura:** Remoção de `useState` globais no `App.tsx` e limpeza das *props* repassadas entre componentes.
- **Otimização de Renderização:** Uso inteligente de seletores para evitar que componentes re-renderizem sem necessidade.

---

## 🚀 Passo a Passo das Tarefas

Abaixo estão os requisitos para você implementar e melhorar no código existente. Cumpra um de cada vez!

### 1. Instalação e Configuração Inicial
- **🎯 Objetivo:** Adicionar o Zustand ao projeto e preparar a estrutura de arquivos.
- Abra o terminal do seu projeto e instale a biblioteca executando: `npm install zustand` (ou `yarn add zustand`).
- Crie uma pasta chamada `store` (ou `stores`) dentro do diretório do seu código fonte (ex: `src/store`).
- Crie um arquivo chamado `useTaskStore.ts` para ser o nosso gerenciador principal de tarefas.

### 2. Criando a Store do Zustand
- **🎯 Objetivo:** Definir o estado global e as ações que irão manipulá-lo de forma centralizada.
- Dentro de `useTaskStore.ts`, importe a função `create` da biblioteca `zustand`.
- Se estiver usando TypeScript, defina a `interface` ou `type` do seu estado (ex: `TaskState`), tipando a lista de tarefas (`tasks: Task[]`) e as funções de ação (como `addTask`, `toggleTaskCompleted`, `deleteTask`).
- Implemente a store com a função `create`. Transfira toda a lógica de manipulação de tarefas que antes ficava no `App.tsx` para dentro das actions da store.
- **Dica Mestre:** Para atualizar o estado no Zustand baseado no estado anterior, você pode usar a função `set` recebendo um callback: `set((state) => ({ tasks: [...state.tasks, newTask] }))`.

### 3. Refatorando os Componentes (O Fim do Prop Drilling)
Agora que temos uma fonte única de verdade (a store), não precisamos mais passar dados de pai para filho!

- **🎯 Objetivo:** Conectar os componentes à nova store global e limpar o código legado.
- Vá no seu `App.tsx` e **remova** a declaração do `useState` que guardava as tarefas.
- Remova todas as propriedades (*props*) relacionadas às tarefas ou funções de alteração que você passava para os componentes filhos (`<TaskList />`, `<TaskItem />`, modais de adição, etc). Eles não precisarão mais receber isso por parâmetro!
- Em cada componente que precisa ler a lista de tarefas ou disparar uma ação (como um botão de excluir), importe o hook `useTaskStore`.
- **Regra de Ouro da Performance:** Utilize **seletores** para extrair apenas o que aquele componente específico precisa. Em vez de pegar a store inteira, faça: `const tasks = useTaskStore((state) => state.tasks)` ou `const deleteTask = useTaskStore((state) => state.deleteTask)`.

### 4. Bônus Avançado: Persistência Local (AsyncStorage)
O que acontece com as tarefas quando fechamos o aplicativo? Elas somem! Vamos resolver isso com Zustand.

- **🎯 Objetivo:** Salvar as tarefas automaticamente no armazenamento local do dispositivo.
- O Zustand possui *middlewares* nativos muito poderosos. Pesquise na documentação oficial sobre o middleware `persist`.
- Instale a biblioteca do React Native para armazenamento: `npx expo install @react-native-async-storage/async-storage`.
- Envolva a criação da sua store no middleware `persist` e configure a propriedade `storage` para utilizar o `AsyncStorage` com o wrapper `createJSONStorage`. 
- Teste fechar e abrir o aplicativo para ver a mágica da persistência acontecendo sem nenhum `useEffect` complexo no seu componente!
