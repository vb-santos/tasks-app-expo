# 📝 Desafio Prático: Autenticação com Signup, Login e Logout

Seja bem-vindo(a) a mais um desafio prático de React Native! O objetivo desta atividade é evoluir o nosso aplicativo implementando um fluxo completo de autenticação. Vocês criarão as telas de Cadastro (Signup) e Entrar (Login), e garantirão que a sessão do usuário permaneça salva no dispositivo utilizando um `sessionToken`.

Como referência visual e de código, utilizem a demo do projeto apresentada em aula: [Habit Tracker Expo](https://github.com/renanalencar/habit-tracker-expo).

## 🎯 Tópicos Abordados

Neste exercício, você terá contato direto com os seguintes conceitos fundamentais:

- **Fluxo de Autenticação:** Construção das interfaces e lógica para criação de conta e acesso ao app.
- **Gerenciamento de Sessão:** Uso e armazenamento de um `sessionToken` para manter o usuário logado entre as sessões.
- **Navegação Protegida:** Criação de rotas que só podem ser acessadas por usuários autenticados.
- **Integração de Formulários:** Captura e validação de dados de entrada do usuário.

---

## 🚀 Passo a Passo das Tarefas

Abaixo estão os requisitos para você implementar no projeto. Cumpra um de cada vez!

### 1. Preparando as Telas de Autenticação
- **🎯 Objetivo:** Criar a base visual para que o usuário possa se cadastrar e fazer login.
- Crie duas novas telas no seu aplicativo (ex: em `app/login.tsx` e `app/signup.tsx` se estiver usando o Expo Router).
- Na tela de **Signup** (Cadastro), crie um formulário com campos para Nome, E-mail e Senha, além de um botão para "Criar Conta".
- Na tela de **Login** (Entrar), crie um formulário com campos para E-mail e Senha, e um botão para "Entrar".
- Adicione links em cada uma das telas para permitir a navegação entre elas (ex: "Não tem uma conta? Cadastre-se").

### 2. Implementando o Fluxo de Signup e Login
- **🎯 Objetivo:** Conectar a interface do usuário com a lógica de autenticação.
- Baseado na demo do repositório ([Habit Tracker Expo](https://github.com/renanalencar/habit-tracker-expo)), implemente as funções chamadas ao submeter os formulários.
- Para o **Signup**, envie os dados do usuário para o seu backend/serviço e trate a resposta de acordo.
- Para o **Login**, ao enviar as credenciais com sucesso, você deve receber um `sessionToken` (ou token de acesso equivalente) da sua API.

### 3. Persistindo a Sessão do Usuário
- **🎯 Objetivo:** Não obrigar o usuário a fazer login toda vez que abrir o aplicativo.
- O `sessionToken` recebido no passo anterior é a chave para o acesso. Salve esse token no armazenamento local do dispositivo.
- Você pode utilizar bibliotecas como o `AsyncStorage` ou `SecureStore` (recomendado para maior segurança) do Expo para armazenar o `sessionToken`.
- Sempre que o aplicativo iniciar, verifique se o `sessionToken` existe no armazenamento local. Se existir e for válido, redirecione o usuário diretamente para a tela principal (área logada). Se não, mantenha-o na tela de Login.

### 4. Implementando o Logout
- **🎯 Objetivo:** Permitir que o usuário encerre a sua sessão de forma segura.
- Adicione um botão de **Sair (Logout)** em alguma parte acessível do seu aplicativo (ex: aba de Perfil ou Configurações).
- Quando o usuário clicar nesse botão, a função correspondente deve **remover o `sessionToken`** do armazenamento local e, opcionalmente, limpar o estado global do usuário.
- Após limpar os dados, redirecione o usuário imediatamente de volta para a tela de Login.

### 5. Rotas Protegidas
- **🎯 Objetivo:** Garantir que o usuário não possa acessar telas internas sem estar logado.
- Configure o seu sistema de roteamento (ex: Expo Router) para verificar o status de autenticação (a presença do `sessionToken`).
- Se um usuário tentar acessar uma rota interna (como a lista de tarefas) sem o token, redirecione-o automaticamente para a tela de Login ou exiba uma mensagem de acesso negado.
